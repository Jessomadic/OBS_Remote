## Project

**OBS Remote** — a Windows server + web UI that gives full remote control of OBS Studio
(scenes, audio, streaming, recording, sources, filters, studio mode, stats) from any browser on the local network.

## Architecture

```
OBS Remote/
├── main.py                     # Top-level entry point (service args or tray+server)
├── version.py                  # Single source of version truth: __version__ = "x.y.z"
├── requirements.txt
├── build.spec                  # PyInstaller build spec
├── version_info.txt            # Windows EXE version metadata
├── server/
│   ├── main.py                 # FastAPI app, WebSocket /ws, lifespan, static UI serving
│   ├── config.py               # JSON config in %APPDATA%/OBSRemote/config.json
│   ├── obs_client.py           # obsws-python v5 wrapper (req/events)
│   ├── updater.py              # Silent auto-updater (GitHub Releases)
│   ├── service.py              # pywin32 Windows Service
│   ├── tray.py                 # pystray system tray icon
│   └── routes/
│       ├── scenes.py           # GET/POST /api/scenes, scene collections
│       ├── audio.py            # GET/POST /api/audio (volume, mute)
│       ├── streaming.py        # GET/POST /api/streaming (stream, record, vcam, replay)
│       ├── sources.py          # GET/POST /api/sources (source visibility)
│       ├── filters.py          # GET/POST /api/filters (filter enable/disable)
│       ├── studio.py           # GET/POST /api/studio (studio mode, transition)
│       └── stats.py            # GET /api/stats (OBS system stats)
├── ui/
│   ├── index.html              # Single-page app shell
│   ├── css/style.css           # Dark purple theme
│   └── js/
│       ├── api.js              # All fetch/WebSocket wrappers (window.api)
│       └── app.js              # Main app init, tab routing, polling, WS event handling
├── assets/
│   └── icon.ico                # App icon (auto-generated by CI if missing)
├── installer/
│   └── setup.iss               # Inno Setup installer script
└── .github/
    └── workflows/
        └── build.yml           # CI: push to main → PyInstaller → Inno Setup → GitHub Release
```

## Stack

| Layer | Technology |
|-------|-----------|
| Language | Python 3.12 |
| Web framework | FastAPI + uvicorn |
| OBS connection | obsws-python (obs-websocket v5, OBS 28+) |
| Windows service | pywin32 |
| System tray | pystray + Pillow |
| Web UI | Vanilla HTML/CSS/JS (no build tools) |
| Packaging | PyInstaller → single folder bundle |
| Installer | Inno Setup → OBSRemote_Setup_x.y.z.exe |
| CI | GitHub Actions (windows-latest runner) |

## Key Config

- Default OBS WebSocket: `localhost:4455` (no password by default)
- Default server port: `42069`
- Config stored at: `%APPDATA%\OBSRemote\config.json`
- GitHub repo for updates: `Jessomadic/OBS_Remote`

## Build Commands

```bash
# Install dependencies (use a venv)
pip install -r requirements.txt
pip install pyinstaller

# Run locally for development
python -m server.main

# Build exe bundle
pyinstaller build.spec --noconfirm --clean

# Build installer (requires Inno Setup installed)
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer\setup.iss
```

## Releasing a New Version

1. Update `version.py` — change `__version__ = "x.y.z"`
2. Commit and push to `main`
3. GitHub Actions automatically:
   - Builds the PyInstaller bundle
   - Creates the Inno Setup installer
   - Creates a GitHub Release tagged `vx.y.z` with the installer attached

The installed app checks for updates hourly and applies them silently.

## OBS WebSocket Requirements

- OBS Studio 28 or newer
- WebSocket server must be enabled: **Tools → WebSocket Server Settings → Enable WebSocket server**
- Default port 4455, no authentication required (or set password in OBS Remote settings)

## Development Notes

- The UI is served as static files from `/ui` by FastAPI — no separate dev server needed.
- WebSocket events from OBS are forwarded to browser clients via the `/ws` endpoint.
- All API routes return JSON. Errors return `{ "detail": "..." }` with HTTP 503.
- The auto-updater runs in a daemon thread; it checks GitHub API and runs the new installer silently with Inno Setup's `/VERYSILENT` flag.
